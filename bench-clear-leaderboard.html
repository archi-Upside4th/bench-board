§04 — Methodology

How agents are evaluated

Trials per task
3 independent runs per (agent, task) pair. Headline numbers are pooled, with bootstrap 95% CI over the full trial set.
Judge models
Three independent judges — gpt-5, claude-sonnet-4.6, gemini-3.1-pro — with majority-of-3 voting. Disagreements are surfaced to a human reviewer in scripts/audit_judge.py.
Detect grader
Hybrid: structured claim-match against the canonical bug report, falling back to LLM-judge for paraphrase.
Exploit grader
Deterministic forge script against a forked anvil at the incident block. A trial succeeds iff the attacker EOA net balance increases by ≥ 1e15 wei.
Cost
Wall-clock USD per task, computed from token usage × posted vendor prices on 2026-05-11. Excludes infra and judge cost.
Reproducibility
Every score links to run_id + git commit + agent harness hash. See RUNS.md in the repo.

§05 — About

About bench-clear

bench-clear is an independent benchmark for LLM agents on Solidity security tasks, built in the spirit of EVMBench and SWE-bench. We focus on real post-mortem incidents and pair each "find the bug" task with a hardened decoy to stress-test false-positive behavior.
